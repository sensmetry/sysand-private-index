# Submitted packages

This folder holds the team's submitted `.kpar` files. See **"Publish a
project"** in the repository README for the step-by-step guide.

A submission is one `.kpar` file (exactly as `sysand build` produced it)
added directly to this folder through a pull request:

```
kpars/my_project-1.0.0.kpar
```

A validation check runs on your pull request and describes what the file
contains in its log. After your submission is approved and merged, the
automation publishes it to the index. Files stay here afterwards: this
folder is the record of what has been submitted. Published versions are
immutable, so never change a file here. Publish a new version instead.
