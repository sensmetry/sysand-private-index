# Published artifacts

This folder holds the team's submitted `.kpar` files — see **"Publish a
project"** in the repository README for the step-by-step guide.

A submission is one `.kpar` file (exactly as `sysand build` produced it)
added directly to this folder via a pull request:

```
kpars/my_project-1.0.0.kpar
```

Validation posts a comment on your pull request describing what the file
contains. After your submission is approved and merged, automation
publishes it to the index. Files stay here afterwards — the folder is the
record of what has been submitted, and published versions are immutable
(never modify a file here; publish a new version instead).
